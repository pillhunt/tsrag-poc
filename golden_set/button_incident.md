### **Почему не с первой попытки**

В `global.log` при тех же сохранениях:

1. **Дубликат ключа в метаданных формы**  
`Violation of PRIMARY KEY constraint 'PK_BlockLines'. Cannot insert duplicate key ... (a72e55e6-7c0f-ee11-80e9-005056a083f2)`  
Типично для повторного сохранения после неуспеха: в БД уже есть строка блока, повторный insert падает.
2. **Deadlock в SQL Server**  
`Transaction ... was deadlocked ... and has been chosen as the deadlock victim`  
Несколько долгих `PutProjectTypeWithBlocks` на одном типе «УО» шли параллельно и блокировали друг друга.
3. **Конфликт версии карточки** (смежно, 20:14)  
«Карточка изменилась. Дождитесь загрузки изменений других пользователей и сохраните карточку еще раз».

Повторные клики «Сохранить» без успешного ответа усиливали дубликаты и deadlock — это объясняет «не с первой попытки» **независимо от браузера**.

**Инцидент:** при добавлении/сохранении новой кнопки в конфигурации типа дела **ОСАГО (УО)** операция `PutProjectTypeWithBlocks` выполнялась **десятки минут**, часто **завершалась ошибкой БД** (дубликат `BlockLines`, deadlock), из‑за чего пользователь делал повторные попытки.

**Не инфраструктура «интернет/браузер»**, а:

- в пике — общая деградация SQL (в т.ч. массовые сохранения убытков ~20:57).
